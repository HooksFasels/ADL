# ADL API Testing Guide (v1)

This guide provides sequential `curl` commands to test the entire lifecycle of the ADL platform. **Follow the order specified** to ensure relational integrity (e.g., you cannot assign a vehicle to a driver if neither exists).

**Base URL**: `http://localhost:3009/api/v1`

---

## 1. College Module

**Step 1: Create a College**

```bash
curl -X POST http://localhost:3009/api/v1/colleges \
-H "Content-Type: application/json" \
-d '{
  "name": "MIT Institute of Technology",
  "domain": "mit.edu",
  "isActive": true
}'
```

*Note: Copy the [id](file:///Users/sunilkumar/ADL/apps/api/src/middlewares/validate.middleware.ts#5-21) from the response as `COLLEGE_ID`.*

**Get All Colleges**

```bash
curl http://localhost:3009/api/v1/colleges
```

## 2. User Module

**Step 2: Create a Driver User**

```bash
curl -X POST http://localhost:3009/api/v1/users
-H "Content-Type: application/json"
-d '{

"password": "securePassword123",
"name": "John Doe",
"role": "DRIVER",
"collegeId": "37efdff0-579f-4880-a0f3-c832862c003a"
```

## 8. Location Module

**Step 9: Update Live Location**

```bash
curl -X POST http://localhost:3009/api/v1/location/update \
-H "Content-Type: application/json" \
-d '{
  "vehicleId": "e0bdd946-38f9-4235-a98c-8b400ddff607",
  "latitude": 40.7128,
  "longitude": -74.0060,
  "speed": 65.5
}'
```

*Note: This will automatically link to the active `TRIP_ID` if one is running for this vehicle.*

**Get Vehicle Location History**

```bash
curl http://localhost:3009/api/v1/location/vehicles/e0bdd946-38f9-4235-a98c-8b400ddff607
```

## 🛠️ Debugging Tips

1. **Health Check**: `curl http://localhost:3009/health`
2. **UUIDs**: All IDs must be valid UUIDs generated by the system in previous steps.
3. **Roles**: User needs a `DRIVER` role to have a Driver Profile.
4. **Dates**: Use ISO 8601 format (`YYYY-MM-DDTHH:mm:ssZ`).
